boost::opaque::new_class

Synopsis

// In header: <boost/opaque/new_class.hpp>

template<typename Final, typename UT, 
         typename MetaMixinSeq = boost::mpl::vector0<>, 
         typename Base = base_opaque_type> 
class new_class : public linear_hierarchy::type< MetaMixinSeq, Final, Base > {
public:
  // types
  typedef UT underlying_type;
  typedef unspecified undelying_type_hierarchy;  // MPL sequence of type to which this type is convertible.

};

Description

new_class<> wraps an underlying type providing the regular constructors and copy construction from the underlying type and all the convertibles to the the UT.

Requires:

  • Final must publicly inherit from this class to make it a model of FinalUnderlying

  • UT must be CopyConstructible and Assignable.

  • MetaMixinSeq must be a model of MetaMixinSeq.

  • Base must inherit directly or indirectly from base_opaque_type or be base_opaque_type.

Note(s):

  • Can instances of UT be explicitly converted to instances of OT? Yes, there is an explicit constructor.

  • Can instances of convertible to UT be explicitly converted to instances of OT? Yes, there is an explicit constructor.

  • Can instances of UT be implicitly converted to instances of OT? No, this is one of the main differentiating features of new_class.

  • Can instances of OT be explicitly converted to instances of UT? Yes, there is an explicit conversion operator (if supported by the compiler) or the underlying function.

  • Can instances of OT be implicitly converted to instances of UT? No, this is one of the main differentiating features of new_class.
