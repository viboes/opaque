boost::opaque::private_opaque_class

class private_opaque_class : public boost::opaque::new_class< Final, UT, mpl::push_front< mpl::push_front< MetaMixinSeq, using_explicit_conversion_to_underlying_type_hierarchy< base_private_opaque_type, UT > >::type, inherited_from_underlying< UT, Bool > >::type, Base >

Description

private_opaque_class<> wraps an underlying type providing the regular constructors and copy construction from the underlying type and all the conversions to the the UT inheriting from all the underlying operations.

Requires:

  • Final must publicly inherit from this class.

  • UT must be CopyConstructible.

  • Bool must be a model of Boolean.

  • MetaMixinSeq must be a model of MetaMixinSeq.

  • Base must inherit directly or indirectly from base_opaque_type or be base_opaque_type.

Note(s):

  • Can instances of UT be explicitly converted to instances of OT? Yes, there is an explicit constructor.

  • Can instances of convertible to UT be explicitly converted to instances of OT? Yes, there is an explicit constructor.

  • Can instances of UT be implicitly converted to instances of OT? No, this is one of the main differentiating features of opaque types.

  • Can instances of OT be explicitly converted to instances of UT? Yes, there is an explicit conversion operator (if supported by the compiler) or the underlying function.

  • Can instances of OT be implicitly converted to instances of UT? No, this is one of the main differentiating features of base_private_opaque_type.

private_opaque_class<> is a new_class with the using_explicit_conversion_to_underlying_type_hierarchy<base_private_opaque_type> and inherited_from_underlying<UT> added to the sequence of meta-mixins MetaMixinSeq, so a private_opaque_class inherits from all the operations of the underlying type UT and adds transitive explicit conversions to all the substituables.
